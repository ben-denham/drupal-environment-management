Local Machine vs. Server

  • Production
    • Probably on a server
  • Testing
    • Server for UAT?
    • Automated tests?
    • Continuous integration
      • [GitLab CI](https://about.gitlab.com/gitlab-ci/), [Jenkins](https://jenkins.io/), [Travis](https://travis-ci.org/), [Circle](https://circleci.com/)
  • Development
    • Typically local
    • In the CloudTM?
      • Develop from anywhere
      • Cloud IDEs: [Eclipse Che](http://www.eclipse.org/che/), [Cloud9](https://c9.io/), [etc.](https://www.slant.co/topics/713/~cloud-ides)

Development - Why Virtualise?

  • Pros
    • Isolation
    • Production-like environment
  • Cons
    • Performance overhead
    • Host<->VM communication
  • Other thoughts? Is a [WL]AMP stack good enough?

Containers?

  • In essence, **isolated processes**
  • A bit like lightweight virtual machines
  • Pros
    • Less overhead than a VM
    • Development == Production
  • Cons
    • A lot of churn
    • Some [security](https://access.redhat.com/blogs/766093/posts/1976473) [concerns](https://access.redhat.com/blogs/766093/posts/1976463)
  • See: [Docker](https://www.docker.com/), [Rocket](https://coreos.com/blog/rocket/), [LXC](https://linuxcontainers.org/)
### Have a look at these projects * Virtual Machines * [Vagrant](https://www.vagrantup.com) * [DrupalVM](https://www.drupalvm.com) * [Beetbox](http://beetbox.readthedocs.io) + [Drupal South Talk](https://www.youtube.com/watch?v=Ff7dEadrY18) * Containers * [Official Drupal Docker Image](https://hub.docker.com/_/drupal) * [Docker4Drupal](http://docker4drupal.org) * [Docksal](https://ffwagency.com/blog/announcing-docksal-docker-based-development-environment)

Why automate configuration (DevOps)?

  • Inherently documented
  • Repeat it anytime, anywhere
  • Track changes (version control)

Infrastructure Configuration

  • How do you provision servers for a new site?
  • How do you scale when site load increases?
  • Automation options:
    • [CloudFormation (Amazon)](https://aws.amazon.com/cloudformation/)
    • [Heat (OpenStack)](https://wiki.openstack.org/wiki/Heat)
    • [Terraform](https://www.terraform.io)

System Configuration

  • How do you install or upgrade a system package (e.g. PHP)?
  • How do you edit Apache configuration?
  • Automation options:
    • Configuration Management: [Ansible](https://www.ansible.com/), [Puppet](https://puppet.com/), [Fabric](http://www.fabfile.org/), [Chef](https://www.chef.io/)
    • PaaS: [Pantheon](https://pantheon.io/), [Heroku](https://www.heroku.com/)
    • Containers

Project Configuration

  • How do you install or upgrade a new module?
  • How do you deploy new code?
  • How do you deploy site configuration (e.g. content types)?
  • Automation options:
    • [Drush Make](http://www.drush.org/en/master/make/), [Composer](https://www.drupal.org/docs/develop/using-composer/using-composer-with-drupal)
    • Continuous Deployment (CI tools)
    • [Aegir](http://www.aegirproject.org/)
    • [Environment](https://www.drupal.org/project/environment), [Features](https://www.drupal.org/project/features), [D8 Config Manager](https://www.drupal.org/docs/8/configuration-management/managing-your-sites-configuration)
